# Security Policy

## Supported versions

The active `main` branch is the supported release line for this repository.

## Reporting a vulnerability

Do not open a public GitHub issue for security problems.

Instead:

1. Gather the affected route, file, or feature.
2. Describe reproduction steps and possible impact.
3. Send the report privately to the repository owner.

If credentials or tokens may have leaked:

1. Rotate the affected secret immediately.
2. Remove the exposed value from the codebase and deployment settings.
3. Rebuild and redeploy the affected environments.
